## Essential Reference Files

### Dashboard Reference Files (`Do's/` directory)
**LOCATION:** `/frontend/Do's/`

This directory contains HTML reference files that are the **source of truth** for dashboard page implementation. These files show the exact structure, styling, and functionality expected for each dashboard page.

**BEFORE working on any dashboard-related code:**
1. Check if a reference file exists in `Do's/` for the page you're working on
2. Read the reference file to understand the expected structure
3. Match the HTML structure, CSS classes, and component hierarchy
4. Preserve the original design intent and user experience

**Reference files available:**
- `DashboardHome` - Main dashboard landing page
- `DayTradingRoomDashboard` - Day trading room interface
- `LatestUpdates` - Latest updates page
- `Learning-Center` - Learning center main page
- `Platform-Tutorials` - Platform tutorials section
- `Premium-Daily-Videos` - Premium videos listing
- `Premium-Daily-Videos-Clicked` - Individual video view
- `wwclickememberdashboard` - Member dashboard interactions

See `Do's/README.md` for complete documentation.

## Tech Stack

- **Framework:** SvelteKit 5 with Svelte 5
- **Styling:** Tailwind CSS 4
- **Language:** TypeScript 5
- **Testing:** Playwright for E2E

## Key Principles

1. **Reference First:** Always check `Do's/` directory before implementing dashboard features
2. **Component Reuse:** Use components from `$lib/components` (ui, patterns, layout)
3. **Design Tokens:** Use color tokens defined in `app.css` with `@theme` directive
4. **Performance:** Code-split heavy libraries, lazy load components
5. **Observability:** Track events and use feature flags from `$lib/observability`

## Common Tasks

### Working on Dashboard Pages
1. Check `Do's/[PageName]` for reference HTML
2. Implement in `src/routes/dashboard/[page]/+page.svelte`
3. Use existing components from `$lib/components`
4. Match the structure and styling from reference file

### Adding New Components
1. Place in appropriate category: `ui/`, `patterns/`, or `layout/`
2. Export from category `index.ts`
3. Follow existing component patterns
4. Use design tokens from `app.css`

### API Integration
1. Create API module in `src/lib/api/[domain].ts`
2. Use base API client from `src/lib/api/client.ts`
3. Handle errors consistently
4. Add TypeScript types in `src/lib/types/`
